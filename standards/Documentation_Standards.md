# Documentation Standards

## Code Documentation
- All public APIs, classes, and methods must have doc comments
- Use JSDoc/TSDoc (or language equivalent) format
- Include: description, parameters, return value, examples
- Keep comments explaining "why", not "what"

## README Files
- Project overview and purpose
- Installation instructions
- Quick start guide
- Configuration options
- Build and deploy steps
- Link to full documentation

## API Documentation
- Document all endpoints with request/response schemas
- Include authentication requirements
- Provide example requests and responses
- Use OpenAPI/Swagger for REST APIs
- Keep documentation in sync with implementation

## Architecture Decisions
- Record significant decisions in ADRs (Architecture Decision Records)
- Format: Title, Context, Decision, Consequences
- Store in `docs/adr/` directory
