# Contract Expiration Sequence

**Document ID:** SEQ-006

---

# Purpose

Defines controlled contract expiration.

---

# Trigger

Expiration date reached or administrative termination.

---

# Main Flow

1. Expiration detected.
2. Active obligations verified.
3. Outstanding issues reviewed.
4. Contract closed.
5. Audit completed.
6. Notifications generated.

---

# Rules

Expired contracts become read-only.
